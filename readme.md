# Шпаргалка по работе с GIT

## команды Git
|Команда|Описание|Комментарий|
|----------------|---|---|
|`git init`|инициализация репы в папке|для отмены:  <br>`rm -rf .git`|
|`git status`|выводит статусы:<br>- ветку  <br>- коммиты  <br>- что попадёт в коммит<br><br>показывает файлы со статусом:<br>- staged<br>- modified<br>- untracked||
|`git add %имя_файла%`|добавляет файл в коммит||
|`gid add .`|добавляет в коммит все изменения в папках||
|`git commit`|коммитит изменения.<br>ключ `-m` добавляет комментарий:<br>`git commit -m "комментарий"`<br><br>`--amend` позволяет внести правки в HEAD коммит:<br> 	- новые файлы<br>	- изменения в коммиченных файлах<br> - коммент<br>с ключом `--no-edit` - оставит коммент прежним||
|`git log`|выводит историю коммитов<br><br>`--oneline` выводит сокращенную версию, с порезанным хешом и комментом||
|`git remote`|для работы с удалённой репой||
|`git remote add`|добавляет удалённую репу, связывая с локальной. <br> синтаксис: <br> `git remote add %имя_репы% %URL%`||
|`git remote -v`|выводит вербозный список удаленных реп||
|`git push`|пушит коммит на удаленную репу. <br> для первого пуша нужно указать имена удаленной репы и текущей ветки, например:  `git push -u origin master`||

## HEAD
**HEAD** - самый свежий, "головной" коммит. Файл головы находится в .git и содержит ссылку на файл с хешем коммита

## Hash
**hash/хеш** - идентификатор коммита, созданный по SHA1

## Статусы файлов
- **untracked** - файл находится в директории гита, но не отслеживается
- **tracked** - изменения в файле отслеживаются гитом
- **staged** - файл подготовлен к коммиту путём git add
- **modified** - в файле есть изенения от последнего коммита  
один и тот же файл может иметь несколько статусов, кроме *untracked* - этот статус взаимоисключающий

## Оформление комментов к коммитам
Комменты к коммитам должны быть информативными и соответствовать заранее-оговренным в компании правилам.  
Максимальная длина, отображаемая в git консоли - 72 символа.  
Если issues отслеживаются в гите - можно указать #%номер\_issue%, чтобы гит автоматически привязал коммит к issue
